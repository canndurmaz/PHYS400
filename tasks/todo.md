## TASK 6 — Literature comparison: MEAM & ML vs ASM/MatWeb elastic data  (IN PROGRESS)

### Goal
Scrape experimental elastic data (Young's modulus E, Poisson ratio ν, shear
modulus G, density) for 20–50 basis-covered materials from the ASM Aerospace
Specification Metals mirror (`asm.matweb.com`), feed the *same* composition to
both predictors (ML surrogate + optimized-MEAM LAMMPS sweep), and emit a single
`literature_comparison.json` with per-material errors + aggregate MAE/MAPE.

### Key findings from recon
- `www.matweb.com` → HTTP 403 (anti-bot) — NOT scrapeable, against TOS.
- `asm.matweb.com` (ASM Aerospace mirror) → clean static HTML, only a broken
  TLS *intermediate* (use `verify=False`). Consistent `?bassnum=<ID>` scheme.
- Pages expose: `Modulus of Elasticity`, `Poisson's Ratio`, `Shear Modulus`,
  `Density`, and a `Wt. %` composition table (ranges → midpoints, `Max x`
  impurities, balance element as bare number). Must HTML-unescape (`&nbsp;`).
- 22 verified materials spanning E = 68.9 → 201 GPa (Al alloys, CP-Ti grades,
  304/316/304L stainless, Inco A-286). Coverage ≥96% of mass in the 12-basis.

### Design (src/literature/)
- [ ] `materials.py` — seed bassnum IDs (grouped) + 12-element atomic masses.
- [ ] `matweb_scraper.py` — fetch (verify=False, polite delay) + on-disk HTML
      cache (offline reruns; `--refresh`) + robust parser.
- [ ] `compare.py` — driver: ML `predict_properties` + MEAM subprocess →
      per-material abs/pct error vs lit E & ν → JSON + summary.
      Flags: `--no-meam`, `--refresh`, `--limit N`, `--min-coverage`.
- [ ] `README.md` — usage, provenance, caveats.

### Verification
- [ ] Scraper parses 7075-O exactly (E=71.7, ν=0.33, G=26.9, ρ=2.81).
- [ ] ML path on all materials; MEAM path (background, ~15 min).
- [ ] JSON validates; aggregate MAPE for both models on E & ν.

### Review  (DONE)

+ `html_cache/` + generated `literature_comparison.json`. **25 materials**,

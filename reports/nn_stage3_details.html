Stage 3 — NN cross-term optimization

PHYS 400 · run-id: --perturbations 150 --sampling active

Purpose. This document explains how the NN-based MEAM optimization stage works after the recent additions of (A) DFT formation-energy targets and (C) per-pair signal-weighted regularization. The central question: how does the optimizer learn binary cross-terms for element pairs that barely co-occur in the random-alloy training set?

2. How can the NN learn pairs with no shared composition?

This is the central question. The honest answer: no binary pair in this 12-element system is genuinely composition-free in the training set. Every pair has at least some signal — but signal strength varies by ~300× across pairs.

However, even for the weakest-signal pairs, the optimizer has three additional, composition-independent signals that constrain the cross-term values.

The four signal sources, in order of strength

For a pair (A, B): Source 1 — random-alloy C_ij [varies 300× across pairs] ─────────────────────────────────── Composition-weighted gradient via 140 training alloys. ρ_pair = mean over alloys of 2·f_A·f_B (≈ 0.0001 for Ni-Zn, 0.027 for Cr-Mo). Strong for Cr-Mo (has Mo₆₂Cr₃₈ binary alloy in training set). Negligible for Ni-Zn (max alloy is Al₈₈Ni₆Zn₄). Source 2 — DFT E_form direct target [Option A — uniform across pairs] ─────────────────────────────────── LAMMPS computes E_form for AB at the exact B2 or L1₂ geometry DFT used. Target value = DFT E_form from dft_results.json. Independent of any composition in results.json — depends only on the binary intermetallic geometry, which we synthesize on the fly. Source 3 — DFT-derived initial seed [the prior] ─────────────────────────────────── Each Ec(A,B) starts at (esub_A + esub_B)/2 + E_form_DFT. In normalized parameter space, the seed corresponds to v_opt = 0. Source 4 — Option-C regularization [pulls weak pairs toward seed] ─────────────────────────────────── Loss += 0.01 × w_pair × v² where w_pair = 1 / max(ρ_pair, 1e-3). For weak-signal pairs (w_pair ≈ 10), the regularization dominates → parameter stays near the DFT seed. For strong-signal pairs (w_pair ≈ 0.04), regularization is light → data signal moves the parameter freely.

Concrete walk-through for Ni-Zn (worst-case pair — ρ = 0.0001, no binary intermetallic in training):

  • Source 1: essentially zero. The composition-weighted gradient through the NN surrogate, evaluated at Ni-Zn parameters, is ~250× smaller than for Cr-Mo. Ni-Zn moves negligibly from data alone.
  • Source 2: full strength. We compute LAMMPS Eform(Ni-Zn) for an L1₂ Ni₃Zn cell at amix = (3.52 + 2.66)/2 = 3.09 Å every Phase-1 sample. Target = DFT Eform(Ni-Zn) = +1.6744 eV. This is one row in the 28-row Eform target block and carries the same weight as every other.
  • Source 3: starting Ec(Ni,Zn) = (4.7287 + 0.9642)/2 + 1.6744 = 4.52 eV. v_opt = 0 means "stay at this value".
  • Source 4: w_Ni-Zn ≈ 10. Reg contribution at v=1σ is 10 × 1 × 0.01 = 0.10 — comparable to the data loss at that drift. Pulls v toward 0 unless data has equal force pushing it away.

So Ni-Zn ends up jointly constrained by (a) its DFT Eform target and (b) the L2 prior on its DFT-seed value. Neither requires Ni-Zn to appear in any training alloy.

Honest caveat. The direction of Ni-Zn's optimization is well-constrained. The magnitude isn't, because Option A's Eform target is only sensitive to a particular linear combination of Ec, re, α (the energy of the L1₂ cell at fixed geometry). Three free parameters per pair → one scalar constraint = an under-determined 2-d subspace. The reg term picks out the minimum-norm-drift solution within that subspace.

4. Why the binary Eform targets are valid

Option A only works if the LAMMPS Eform calculation matches DFT's calculation exactly. Otherwise the comparison is apples-to-oranges and the target is meaningless.

Both DFT and LAMMPS use:

  • Same structure: B2 (2-atom AB) when at least one element is BCC or diamond; L1₂ (4-atom A₃B) when both are FCC or HCP-FCC compatible. Mapping in _BINARY_LATTICE, identical in both files.
  • Same lattice parameter: amix = (aexp,A + aexp,B)/2, using experimental a values listed in _ELEMENT_LATTICE. Not the DFT-equilibrium a from dft_results.json — DFT used the experimental a, and we follow suit.
  • Same energy evaluation: single-point at the unrelaxed geometry. No structure relaxation. DFT does one SCF; LAMMPS does one run 0.
  • Same formula: Eform = Emix/natoms − (nA·Epure,A + nB·Epure,B)/natoms.
The pure-element reference energies (Epure,A, Epure,B) are computed by LAMMPS using the same MEAM library, at each element's library alat. DFT's references come from DFT SCF at DFT-equilibrium alat. These two reference energies differ by a constant (the absolute energy scale of the two methods), but the formation energy — being a difference of energies at the same geometry — cancels that constant out. So the comparison is valid up to the limit of how well MEAM reproduces DFT atomic energetics.

5. Phase 3: how the regularization keeps weak-signal pairs honest

Without Option C, the inverse optimization (Phase 3) can drift weak-signal parameters to values that minimize the surrogate's loss but have no physical basis. The surrogate's gradient in those directions is small but nonzero, and over 3000 Adam steps a tiny gradient becomes a meaningful drift.

The math

v_opt = tf.Variable(zeros((1, 84)))         ← in normalized parameter space
                                              v=0 ⇔ initial DFT seed
                                              v=±1 ⇔ ±1σ from training mean

target_norm = (target_vec − y_mean) / y_std ← 308-dim z-scored target

for step in 3000:
    pred = NN(v_opt)
    data_loss = mean((pred − target_norm)²)
    reg_loss  = 0.01 × mean(w_pair × v_opt²)
    loss      = data_loss + reg_loss
    v_opt   −= 0.01 × ∂loss/∂v_opt           ← Adam update
    v_opt    = clip(v_opt, ±2.5)

The weight vector

# Done once at start of Phase 3:
for name in names:                                 # e.g. "par:Ec(9,12)"
    pair = sym_pair(name)                          # → ("Ni", "Zn")
    pair_signal[name] = mean(2·f_A·f_B for train alloys)

raw_w  = 1 / max(pair_signal[name], 1e-3)         # cap so no infinity
w_pair = raw_w / mean(raw_w)                       # normalize: mean(w) = 1

Concretely:

Pairρraw_w = 1/ρnormalized w
Cr-Mo0.027370.04
Al-Mo0.0042770.28
Mg-Ni0.000616661.65
Ni-Zn0.00011000010.0
(mean)—~10001.00

So Phase-3's reg loss at v = (1σ, 1σ, …) ≈ 0.01 (=λ). At v = (Cr-Mo: 1σ, Ni-Zn: 0σ), reg loss ≈ 0.01·0.04/84 ≈ 5×10⁻⁶. At v = (Cr-Mo: 0σ, Ni-Zn: 1σ), reg loss ≈ 0.01·10/84 ≈ 1.2×10⁻³. So Ni-Zn drift is ~240× more "expensive" than Cr-Mo drift, even though both have the same data loss sensitivity in normalized space.

Net effect. The optimizer freely tunes Cr-Mo/Mn-Mo/Co-Mg/Fe-Mo (the pairs you have direct binary alloy data on) toward the joint Cij + Eform optimum. It conservatively pins Ni-Zn / Mo-Ni / Ni-Si near the DFT seed unless the Eform target gradient is genuinely large there.

What to look for

PatternInterpretation
Strong-signal pairs (Cr-Mo, Mn-Mo, …) all < 0.05 eV/atom Surrogate worked well. Option A injected useful info beyond Cij.
Weak-signal pairs (Ni-Zn, Mo-Ni, …) at 0.1–0.4 eV/atom Expected. Reg term traded direct fit for stability near DFT seed. This is the right behavior.
Any pair > 1 eV/atom MEAM functional form bottleneck. Ec/re/α can't reach the DFT target because angular terms (Cmin, Cmax) are frozen. To improve, you'd unfreeze those.
RMS > 0.3 eV/atom Surrogate didn't converge to the joint optimum. Re-run with higher --perturbations or check that Phase-2 training loss bottomed out.
Strong-signal pairs got worse than baseline Cij targets are dominating and Option A's Eform targets are being sacrificed. Increase the Eform entries' relative weight in the y-normalization (advanced tuning).
